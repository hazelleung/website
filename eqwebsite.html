The Equilibrium Website

A website redesign to improve UX and branding.

My Role

UX Designer

Context

Personal Project

Tools Used

HTML, CSS, JS

Opportunity

The Equilibrium is an undergraduate research journal that was recently accepted to obtain an official UC San Diego domain. As a result, I had the opportunity to redesign the website to improve usability and increase the journal's online presence. The original website was hosted with Google Sites, which meant customization difficult and design was limited to specific themes. I utilized HTML, CSS, and JavaScript to rebuild and redesign the journal's website.

Homepage

The homepage was the biggest opportunity to improve branding and usability because it is the first page that visitors see when they go on the website. The old website homepage had minimal information as to what The Equilibrium was. The only content was an events calendar that was empty for more than 90% of the time, which meant that the rest of the page was empty. In the redesign, the visible section of the page is a maroon geometric background in order to create a stronger brand image. Additionally, I included more information about the journal on the homepage: what the journal is, what it does, and how to get involved with the journal.

Application Pages

These secondary pages needed to be reorganized because submitting a manuscript to publish is different from applying to join the journal's editorial board. Thus, I separated out the orignal website's "Application" page to a "Submit" page for authors to learn how to submit their reserach for consideration and a "Apply" page for students to learn how to joing the organiation.

Publication Pages

Another major redeisgn was the publication pages. The original page only had links to the online readers. However, there was a lot of new information to include. Additionally, in order to increase our online presence, each individual publication is accessbile on the website as well.

Reflection

Redesigning an entire website (even if it's a site that only has five pages) took a lot of time. But, I found that I really enjoyed coding and creating a website from scratch. It gave me ample opportunity to practice the HTML, CSS, and JavaScript skills I already had. At the same time, this experience forced me to dig deeper. I had to really understand the neuances of the coding languages I was using, to find what the best practices are, and to discover how to implement specific designs in order to make a website that was aesthetic and functional. This process was a practice of trial, error, and Googling, which has, ultimately, made me a better designer.
